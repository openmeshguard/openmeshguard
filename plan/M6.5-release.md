# M6.5 — v0 Release Engineering

Branch: `m6.5-release`

## Goal
Cut `v0.1.0` once M6c lands: reproducible signed release artifacts, an install path beyond `go install`, and the first-impression docs a public download deserves. No product behavior changes.

## Context
Triggered by M6c completion (the DoD "first-run zero-config scan produces the SPEC §6-shaped summary with honest unknowns" is the release criterion). The `go install` path and ldflags version stamping already exist (PR #6); this milestone adds the binary-distribution tier.

## Deliverables
- [ ] goreleaser config: darwin/linux × amd64/arm64 archives, `-ldflags "-X main.version="` wired to the tag, checksums file. Reuse the existing `VERSION=` make override.
- [ ] Release workflow (tag-triggered) building and publishing GitHub Release artifacts; cosign signing of archives + checksums.
- [ ] Verify the e2e goldens that pin `scanner.version: "dev"` are unaffected by release builds (release stamps a real version; `make build` still yields `dev`) — or scrub the field. Confirm before tagging.
- [ ] README: promote the alpha `go install` line to a real install section (release binaries + `go install`), accurate quickstart matching shipped commands, honest "what works / what's next" table.
- [ ] SECURITY.md (disclosure policy — this is a security tool), CHANGELOG.md for v0.1.0, and a short release note framing v0 as declared+resolved posture with runtime verification (M7) as the headline still to come.
- [ ] Optional: Homebrew tap formula, since the audience is platform/security engineers who won't all have a Go toolchain.

## Definition of Done
- `v0.1.0` tagged; release artifacts built, signed, checksummed, and downloadable; a downloaded binary runs `openmeshguard version` reporting the tag and completes a scan.
- README install path verified from a clean machine (both `go install @v0.1.0` and a release binary).
- No product/scan behavior changed vs. M6c; `make build test lint schema-test` + e2e green.

## Out of scope
Any scanner feature work, M7 runtime verification.
